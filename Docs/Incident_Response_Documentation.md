# Incident Response 

## 1. What is Incident Response?
Incident Response (IR) is a structured process used to detect, analyze, contain, eradicate, and recover from cybersecurity incidents.

## 2. Phases of Incident Response
1. Preparation
2. Identification
3. Containment
4. Eradication
5. Recovery
6. Lessons Learned

## 3. Role of Logs in Incident Response
Logs provide evidence of attacker activity such as:
- Failed login attempts
- Unauthorized access
- Privilege escalation
- Malware execution

## 4. Root Cause Analysis
Root cause analysis determines how the incident occurred so that the same attack cannot happen again.

## 5. Importance of Incident Response
- Reduces attack impact
- Minimizes downtime
- Protects sensitive data
- Improves security defenses
