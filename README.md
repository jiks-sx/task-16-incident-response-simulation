# Incident Response & Security Breach Simulation

## Objective
Simulate a security incident, analyze system logs, and perform incident response steps to detect, contain, and remediate the threat.

## Tools Used
- Linux authentication logs
- Windows Event Viewer (optional)
- Basic incident response workflow

## Workflow
1. Simulate security incident
2. Analyze logs for suspicious activity
3. Classify incident severity
4. Contain affected systems
5. Remove threat and root cause
6. Restore secure operations
7. Document incident timeline
8. Recommend improvements
